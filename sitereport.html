Validation Reports

For web pages to be of high quality and well-performing, it is necessary to validate HTML and CSS code. I checked my code using W3C Markup Validation Service and CSS validator. Some errors were disclosed during the validation process; for example, some alt attributes were missing, or else wrong CSS syntax was used. These problems contributed towards enhancing overall code quality as well as compliance with web standards. The validation reports made clear where improvement was needed. The procedure has been illustrated by providing screenshots of the validation results. At last, user experience and accessibility have been accomplished through checking codes thereby making best practices in web development imperative.

Design Decisions

Designing the website involved several crucial decisions to enhance user experience and visual appeal.

The website employs the 'Poppins' font for a clean, modern look and uses global styles to ensure consistency across browsers. The fixed navigation bar features hover effects for a dynamic user experience. The flex layout adapts to various screen sizes, with distinct background gradients enhancing visual separation. Icons have hover effects for visual feedback, and content styling highlights key information. Circular elements with images and shadows add depth. Media queries ensure responsiveness, with a hamburger menu for mobile navigation. The projects and contact sections use CSS Grid and Flexbox for a polished appearance, and the site report includes expandable content and an image gallery for validation.
